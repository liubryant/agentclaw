# ChatSearchActivity 弹窗化改造方案

## Summary
将 `ChatSearchActivity` 保持为独立 `Activity`，但视觉和交互改造成“全屏透明宿主 + 中央悬浮搜索面板”的弹窗形态，外层加半透明遮罩，面板尺寸、圆角、留白和搜索框样式对齐 Figma `2625:6739`。  
搜索结果改为“按时间段分组的无图标列表”，分组规则复用 `ChatSessionAdapter` 的 `今天 / 本周 / 本月 / 更早`，组内按时间倒序展示。点击任一结果只进入对应会话，不再传递或处理消息级定位。

## Key Changes
- 视觉结构
  - 给 `ChatSearchActivity` 单独使用透明主题，窗口背景透明，内容层由遮罩层 + 白色圆角面板组成。
  - 面板内顶部结构按 Figma 重做：标题“搜索”、右上关闭按钮、圆角搜索框；不保留现有左侧返回按钮布局。
  - 搜索框右侧增加清空按钮；输入中和有内容时的状态切换与截图一致。
  - 面板外区域支持点击关闭，系统返回键同样关闭；软键盘仍保持 `adjustResize`。

- 结果列表与数据映射
  - 移除当前“会话 / 消息 / 工具调用”三类分组头展示方式，改为统一结果流。
  - 保留三类命中来源，但全部映射为统一的 `SearchResultUiItem`：
    - `DateHeader`
    - `ResultRow`
  - `ResultRow` 字段固定为：主标题、摘要文本、日期文本、会话 key、命中时间戳、来源类型。
  - 会话标题命中也走统一行样式，不单独做卡片样式。
  - 结果分组逻辑直接复用 `ChatSessionAdapter` 的时间分桶规则，避免两套时间分段标准不一致。
  - 组内排序按命中时间倒序；同一会话下的多条命中保留，不去做二次折叠。
  - 日期展示从当前 `DateUtils.getRelativeTimeSpanString()` 改成截图式绝对日期，格式固定为 `yyyy/MM/dd`。

- 列表 item 样式
  - 去掉所有图标、类型 badge、卡片背景。
  - 每行使用“标题/摘要/日期”三段式：
    - 第一行左侧主标题，右侧日期。
    - 第二行展示摘要，允许 2 行左右截断。
    - 行间使用分隔线或间距，整体更接近截图而不是卡片列表。
  - 日期头使用弱化样式，和结果行拉开层级，但不做大面积占位。

- 搜索高亮
  - 继续利用当前 snippet 的 HTML 高亮能力，但统一高亮颜色、字重、去掉默认链接/下划线影响。
  - 对会话标题命中补充本地高亮逻辑，避免只有消息 snippet 高亮、标题不高亮。
  - 高亮策略固定为：仅高亮用户输入的完整查询文本；大小写不敏感；空白裁剪后参与匹配。
  - 如果 DAO 返回的 snippet 高亮样式与设计不一致，在 UI 层二次转成 `Spannable`，统一颜色值和字重。

- 跳转与状态
  - 点击结果后只调用 `openShell(sessionKey, null)`，不再传 `messageIndex`。
  - `Empty / Loading / NoResults / Error` 四种状态保留，但视觉改为面板内部状态区，不占满整个屏幕。
  - 空态文案继续沿用当前资源；如需更贴近设计，可只调整排版不改语义。
  - 启动入口仍由 `ShellActivity` 打开 `ChatSearchActivity`，不改导航入口位置。

## Public APIs / Interfaces
- `ChatSearchAdapter`
  - 从当前多 ViewType 的“按来源分组”模型，改为“按日期分组”的统一 UI 模型。
  - 新增统一结果行数据结构，旧的 `Header/Session/Content` 列表构造逻辑整体替换。
- `ChatSearchActivity.openShell(...)`
  - 调用约定调整为只传 `sessionKey`，搜索页不再依赖消息定位参数。
- 资源层
  - 新增弹窗容器布局/背景/分割线/关闭按钮等资源。
  - `ChatSearchActivity` 增加专用 theme，避免污染全局页面 theme。

## Test Plan
- UI 交互
  - 打开搜索页后应显示半透明遮罩和圆角面板，点击遮罩或关闭按钮可退出。
  - 输入少于 2 个字符时显示空态，不触发结果列表。
  - 输入有效关键词后显示加载，再显示结果或无结果态。
  - 点击任意结果后进入对应会话，不再尝试定位到具体消息。
  - 清空输入后恢复初始空态。

- 列表展示
  - 结果按 `今天 / 本周 / 本月 / 更早` 正确分组。
  - 同组内按时间倒序。
  - 无图标、无类型 badge、无旧卡片背景残留。
  - 日期格式固定为 `yyyy/MM/dd`。
  - 长标题/长摘要能正确省略，不挤压日期。

- 高亮
  - 消息命中和会话标题命中都能高亮搜索词。
  - 中英文混合、大小写差异、带空格关键词时高亮行为稳定。
  - 无结果或空摘要时不会出现裸 HTML 标签或样式错乱。

## Assumptions
- 采用“保留 `Activity`、只做弹窗化外观”的方案，不改成 `DialogFragment`。
- Figma 使用 MiSans，但项目当前没有字体资源；实现阶段先用系统 `sans-serif / sans-serif-medium` 做等价降级，不单独引入字体文件。
- 搜索数据源继续复用现有 DAO/Repository，不新增数据库字段；仅在 Repository 或 Adapter 层重组 UI 数据。
- 会话标题命中、消息命中、工具调用命中都继续展示，只是统一成同一种结果行视觉。
